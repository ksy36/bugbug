OTP cell: input[type=number] with appearance:none

What should happen: six narrow (25px) <input type="number"> cells styled with appearance:none. The site uses appearance:none intending to remove the native number spin-button so a single typed digit fills each tiny cell.

Chrome: honours appearance:none and removes the spin-button, so each cell keeps a content box (clientWidth > 0) and the typed digit "8" is visible and centered.

Firefox: does not remove the input[type=number] spin-button for appearance:none (only appearance:textfield / -moz-appearance:textfield does that). The spin-button is laid out inside the 25px cell and consumes the whole content box, so clientWidth === 0: the cell shows only up/down spinner arrows ("drop-down") and the digit has no room to render.

Row A — appearance:none (like the real site)

Row B — control: same width, appearance:textfield

Row C — control: type=text (no spin-button at all)
